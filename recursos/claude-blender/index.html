01 · Qué cambia de verdad

Claude deja de hablar sobre Blender y empieza a mirar tu escena abierta.

El anuncio de Claude lo muestra bien: leer la escena, debuggear, crear tools nuevas o aplicar cambios en batch sin salir del chat.

Según la guía oficial de Anthropic, el conector le da a Claude acceso a tu escena abierta a través de la Python API de Blender. Eso habilita tres cosas muy concretas: entender setups complejos, hacer cambios repetitivos en muchas piezas y escribir herramientas nuevas para tu proyecto.

No es un viewer web ni una importación mágica del archivo. Claude actúa sobre la escena que tenés abierta en esa misma máquina. Cuando guardás en Blender, los cambios recién ahí quedan escritos en disco.

MODELO MENTAL

No le pasás un `.blend` a Claude.
Conectás Claude al Blender que ya está corriendo en tu compu.

02 · Qué necesitás

Cuatro cosas antes de empezar a probar prompts.

01
Claude Desktop
La guía oficial de Anthropic lista Claude Desktop como requisito y aclara que funciona con cualquier plan, incluido Free.
02
Blender 4.2 o superior
Es el mínimo que marca el tutorial de Anthropic para el conector oficial.
03
La escena abierta localmente
No funciona desde el navegador. Anthropic lo deja explícito en el FAQ: el conector necesita Blender y Claude Desktop corriendo en la misma máquina.
04
Guardá una copia antes de pedir cambios masivos
El repo open source `blender-mcp` avisa que Claude puede ejecutar Python dentro de Blender. Úsalo como una herramienta potente, no como sandbox inocuo.
03 · Setup oficial

El paso a paso que hoy sí coincide con la documentación.

Paso 1
Agregá Blender en Claude Desktop

Andá a Customize > Connectors, buscá Blender y apretá Add.

Paso 2
Instalá el add-on dentro de Blender

Abrí la página Blender MCP Server, arrastrá el link de instalación a Blender, aceptá el repositorio de lab y arrastrá ese mismo link una segunda vez para instalar el add-on.

Paso 3
Conectalo desde tu escena

En el 3D Viewport apretá N, abrí la pestaña BlenderMCP y elegí Connect to Claude. En Claude vas a ver las tools bajo el icono del conector.

Detalle raro pero real El doble drag del link no es un invento: primero agrega el repositorio de Blender Lab y después instala el add-on. Es el punto menos obvio de todo el setup oficial.
04 · Qué pedirle primero

Cinco prompts para probarlo sin caer en el clásico "haceme algo lindo". 

Mirá la escena abierta y renombrá objetos, materiales y colecciones para que cada nombre describa lo que realmente contiene.
Explicame paso a paso el Geometry Nodes del objeto activo y marcá qué parte convendría simplificar antes de tocar nada.
Listá todos los objetos que usan este material y decime qué se rompería si lo elimino.
Decime cuáles son los meshes más pesados de la escena y cuáles ocupan poco cuadro en la cámara activa.
Creá una luz tipo estudio, apuntá la cámara al objeto principal y dejá una versión isométrica limpia para preview.
MEJOR PRÁCTICA

Pedile primero que lea y explique. Después pasá a pedir cambios. Ese orden baja muchísimo la chance de que improvise una solución sin entender la lógica de tu archivo.

05 · Límites y atajos

Lo que conviene saber antes de usarlo como si fuera magia.

→ No corre en claude.ai.

Anthropic lo aclara en el FAQ: necesita Blender en la misma máquina, así que hoy el camino real es Claude Desktop.

→ Cambia la escena abierta, no una copia aparte.

Si querés aprender o auditar, pedile que te muestre el plan o el código antes de ejecutar. La guía oficial dice que no hace falta saber Python, pero podés pedir verlo igual.

→ Si falla, reiniciá y partí la tarea.

El README de `blender-mcp` recomienda reiniciar Claude y la conexión de Blender si algo se cuelga, y dividir pedidos largos en pasos más chicos.

→ Hay un camino manual para Claude Code.

Si no querés usar Claude Desktop, la propia guía de Anthropic deriva al setup MCP de Blender y al manual de Claude Code. Para la mayoría, igual, el flujo visual de Desktop es el mejor punto de partida.
